# M3C Progress Log

## Milestone: Discourse User API Key Authentication

Branch: ralph/m3c-discourse-auth

---

## Overview

This milestone implements the Discourse User API Key authentication flow,
mirroring the approach used by the official Discourse mobile app.

Background: Bearer token passthrough from Keycloak SSO does not work with
Discourse. Discourse expects either a session cookie (web) or a User API Key
(mobile apps). See OPEN_QUESTIONS.md Q-002 for details.

Reference implementation: https://github.com/discourse/DiscourseMobile

## Notes

- The official Discourse app uses RSA 2048-bit keys
- Scopes requested: notifications, session_info (minimal read-only initially)
- Custom URL scheme: piraten://discourse-auth
- User API Key header: User-Api-Key (not Authorization: Bearer)
