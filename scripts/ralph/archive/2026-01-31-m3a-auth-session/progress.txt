## Codebase Patterns

### OIDC Discovery Pattern
- Use `OIDAuthorizationService.discoverConfiguration(forIssuer:)` for OIDC discovery
- Wrap callback-based AppAuth API in `withCheckedThrowingContinuation` for async/await
- Access additional endpoints via `configuration.discoveryDocument` (userinfo, jwks)
- Map AppAuth types to domain models to avoid leaking framework types

### AppAuth Authorization Flow Pattern
- Use `OIDAuthState.authState(byPresenting:presenting:)` for authorization code flow
- AppAuth handles PKCE automatically (code_challenge/code_verifier generation)
- Store `OIDExternalUserAgentSession` reference during auth flow for redirect handling
- Handle redirect URL via `.onOpenURL` in SwiftUI with `resumeExternalUserAgentFlow(with:)`
- Map OIDErrorCode.userCanceledAuthorizationFlow to domain AuthError.cancelled

### iOS Keychain Token Storage Pattern
- Use `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` for device-locked, non-synced token storage
- Delete existing item before SecItemAdd to handle overwrites (errSecDuplicateItem)
- Store token expiration as Unix timestamp string for easy parsing
- Check expiration with 60-second grace period before considering token invalid
- Fall back to refresh token presence if access token is about to expire

### AppAuth Token Refresh Pattern
- Use `OIDAuthorizationService.perform()` with an `OIDTokenRequest` for manual token refresh
- Build token request with grantType `OIDGrantTypeRefreshToken` and the stored refresh token
- Handle OAuth token errors (OIDOAuthTokenErrorDomain) as session expiration requiring re-auth
- Refresh tokens may or may not be rotated by server - preserve new refresh token if returned
- Provide `getValidAccessToken()` method that auto-refreshes when token expires within 60s

### Authenticated HTTP Client Pattern
- Use protocol abstraction (`HTTPClient`) to decouple from URLSession implementation
- Wrap base client with `AuthenticatedHTTPClient` for Bearer token injection
- `TokenProvider` protocol allows flexible token retrieval (from AuthStateManager, mock, etc.)
- Map HTTP status codes to domain errors: 401→unauthorized, 403→forbidden, 404→notFound
- Use `onAuthError` callback for 401/403 to trigger re-auth without infinite loops
- Check `response.isAuthenticationError` property for quick auth failure detection
- Use `StubHTTPClient` for testing with configurable URL→response mappings

### OIDC Logout Pattern
- Prefer local-only logout (token deletion) for mobile apps over RP-Initiated logout
- Clear all token types from Keychain: access_token, refresh_token, id_token, expiration
- Use `try?` for token deletion to avoid errors on missing items
- Reset auth state to `.unauthenticated` after token cleanup
- Document end_session_endpoint decision in DECISIONS.md (use vs not use)

### Documentation Pattern
- Use Docs/DECISIONS.md to record architectural decisions with date, status, context, decision, and rationale
- Use Docs/OPEN_QUESTIONS.md to track blocking questions with status, blocking milestone, and what's needed
- Mark questions as "Partially Implemented" when work has proceeded with assumptions
- Reference RFC numbers and official specs in decisions for credibility
- Include configuration tables for quick reference of auth parameters
