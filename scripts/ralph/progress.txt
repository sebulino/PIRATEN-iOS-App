## Codebase Patterns

### Domain Model Pattern
- Domain models live in `PIRATEN/Core/Domain/{Feature}/`
- Use `Identifiable` and `Equatable` protocols for SwiftUI compatibility
- Keep domain models independent of API JSON shape - DTOs handle mapping in the data layer
- Use descriptive comments referencing API concepts without coupling to them

### Repository + ViewModel Pattern
- Repository protocols define async data access in `PIRATEN/Core/Domain/{Feature}/`
- Fake implementations live in `PIRATEN/Core/Data/{Feature}/`
- ViewModels live in `PIRATEN/App/ViewModels/`
- All repository methods should be `async` to support both in-memory and future network implementations
- Use `@Published` properties in ViewModels for SwiftUI state binding
- Wire dependencies through AppContainer as the composition root
- Pass ViewModels through view hierarchy via initializers (not environment objects)
