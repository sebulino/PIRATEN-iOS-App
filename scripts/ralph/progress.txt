### Discourse API Client Architecture
- Use `DiscourseAPIClient` for all Discourse API calls (not the repository directly)
- API client returns raw `Data` for decoding flexibility; repository maps to domain models
- Error handling: `DiscourseError` provides German localized messages for UI display
- Auth: Bearer token injected via `AuthenticatedHTTPClient`; no explicit token handling in API client

- **Context7 / Web Research Summary:**
  - Discourse API supports authentication via HTTP headers (not just URL params)
  - Standard API keys use `Api-Key` + `Api-Username` headers
  - User API keys use `User-Api-Key` header
  - OAuth2 bearer tokens use `Authorization: Bearer <token>` header
  - Rate limits for authenticated users: 20 requests/minute, 2,880 requests/day
  - Source: https://meta.discourse.org/t/api-can-i-authenticate-without-putting-the-key-in-the-url/29425
  - Source: https://meta.discourse.org/t/user-api-keys-specification/48536
