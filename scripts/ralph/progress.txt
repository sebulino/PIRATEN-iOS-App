## Codebase Patterns

### Discourse API Client Architecture
- Use `DiscourseAPIClient` for all Discourse API calls (not the repository directly)
- API client returns raw `Data` for decoding flexibility; repository maps to domain models
- Error handling: `DiscourseError` provides German localized messages for UI display
- Auth: Bearer token injected via `AuthenticatedHTTPClient`; no explicit token handling in API client

### Auth Error Handling (M3B-006)
- Single-attempt rule: `AuthStateManager.isHandlingAuthError` flag prevents concurrent 401/403 handling
- First auth error triggers logout + `.sessionExpired` state; subsequent errors ignored
- Flag reset on: successful re-authentication OR explicit logout
- No automatic retry of token refresh - user must explicitly re-authenticate
- `SessionExpiredView` provides clear "Sitzung abgelaufen" message with login button
