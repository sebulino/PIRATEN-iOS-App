## Codebase Patterns

### Domain Model Pattern
- Domain models live in `PIRATEN/Core/Domain/{Feature}/`
- Use `Identifiable` and `Equatable` protocols for SwiftUI compatibility
- Keep domain models independent of API JSON shape - DTOs handle mapping in the data layer
- Use descriptive comments referencing API concepts without coupling to them

### Repository + ViewModel Pattern
- Repository protocols define async data access in `PIRATEN/Core/Domain/{Feature}/`
- Fake implementations live in `PIRATEN/Core/Data/{Feature}/`
- ViewModels live in `PIRATEN/App/ViewModels/`
- All repository methods should be `async` to support both in-memory and future network implementations
- Use `@Published` properties in ViewModels for SwiftUI state binding
- Wire dependencies through AppContainer as the composition root
- Pass ViewModels through view hierarchy via initializers (not environment objects)

- **Context7 Mobile Auth Security Learnings (OWASP MASTG + Cheat Sheets):**
  - Use `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` for Keychain items (prevents backup extraction)
  - Consider `kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly` for items requiring device passcode
  - Biometric protection via `SecAccessControlCreateFlags.biometryCurrentSet` invalidates on biometric changes
  - Refresh tokens should use same Keychain protections as access tokens
  - Token transmission: TLS 1.2+ required, tokens in Authorization header only (never in URLs)
  - Token lifetime management: short-lived access tokens + secure refresh token rotation
  - OIDC JWT validation: verify signature, iss, aud, exp, iat claims
- **Learnings for future iterations:**
  - Thin-client model: no persistent caching of sensitive content (forum posts, messages, tasks)
  - Trust boundary documentation helps clarify security responsibilities
  - Open questions should include detailed protocol requirements (grant types, PKCE, token rotation)
