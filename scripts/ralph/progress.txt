## Codebase Patterns

### Message Safety Service Pattern (M4-003)
- Use a dedicated `MessageSafetyService` for rate limiting and validation
- Constants in `MessageSafetyConstants` enum for easy auditing
- Validation returns typed enum (`MessageValidationResult`) with error messages
- Rate limiting: track `isSendInProgress` and `lastSendTime` for cooldown
- Never log message content, user identifiers, or any PII in safety code
- Service is `@MainActor` isolated for UI integration
- Character count display: show when content exceeds 50% of max length

### Reply Composer UI Pattern (M4-001)
- Use `@FocusState` for managing text field focus programmatically
- Place composer at bottom of VStack with main content using `.frame(maxWidth: .infinity, maxHeight: .infinity)`
- Use `ReplyComposerState` enum to track idle/sending/sent/failed states
- Disable controls during sending state, show progress indicator on send button
- Use `TextField` with `axis: .vertical` and `lineLimit(1...6)` for expandable text input
- Keep error/success banners above the input area with dismiss actions
