## Codebase Patterns
- **SwiftUI State Management**: Use `@StateObject` with initializer for dependency injection: `_authStateManager = StateObject(wrappedValue: AuthStateManager(authRepository: authRepository))`
- **Domain Layer**: Protocols live in `Core/Domain/`, implementations in `Core/Data/`
- **Auth States**: Use explicit enum states: `unauthenticated`, `authenticating`, `authenticated`, `failed(AuthError)`
- **Typed Errors**: Domain errors should be enums with `Error` conformance and localized descriptions
- **Credential Storage**: Use `CredentialStore` protocol; `KeychainCredentialStore` for production, `InMemoryCredentialStore` for tests/previews

- **AppContainer Pattern**: Use a central `AppContainer` class as the composition root. Initialize at app entry point and pass to views via constructor injection.
- **Constructor Injection in SwiftUI**: Use `let container = AppContainer()` in the `App` struct, then pass `container.authStateManager` to views
