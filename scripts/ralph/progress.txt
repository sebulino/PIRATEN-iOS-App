## Codebase Patterns
- Use `LogRedactor` (Core/Support/LogRedactor.swift) for all logging of sensitive values — tokens, URLs with params, data payloads
- Apple `os.log` Logger DEBUG/INFO are NOT privacy-redacted by default; treat them as public
- Prefer logging metadata (lengths, counts, prefixes) over actual values

- **Context7 summary (URLSession caching & network resilience):**
  - `URLCache` supports both memory and disk caching; `useProtocolCachePolicy` respects HTTP Cache-Control/ETag/Last-Modified headers
  - `waitsForConnectivity` on URLSessionConfiguration waits for network availability instead of failing immediately
  - Apple's new HTTP loader (iOS 18+) supports HTTP/2 and HTTP/3 natively; no special configuration needed
  - Retry logic should be bounded and use exponential backoff; only safe for idempotent (GET) requests
  - Swift Foundation's `URLRequest.CachePolicy` enum provides fine-grained cache control per request
  - `URLCache(memoryCapacity:diskCapacity:)` is the modern initializer; `diskPath:` is deprecated
- **Learnings for future iterations:**
  - TodoRepository was swallowing all errors by returning empty arrays — always make fetch methods `throws` so errors reach the UI
  - `waitsForConnectivity: true` is valuable for mobile — prevents immediate failure on poor networks
  - Only retry GET requests to avoid duplicate mutations
